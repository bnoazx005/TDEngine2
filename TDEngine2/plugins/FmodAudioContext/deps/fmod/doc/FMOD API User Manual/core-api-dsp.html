7. Core API Reference | DSP

The Digital Signal Processor is one node within a graph that transforms input audio signals to an output stream.

Create with System::createDSP, System::createDSPByType or System::createDSPByPlugin.

Connections:

  • DSP::addInput Adds a DSP unit as an input to this object.
  • DSP::getInput Retrieves the DSP unit at the specified index in the input list.
  • DSP::getOutput Retrieves the DSP unit at the specified index in the output list.
  • DSP::getNumInputs Retrieves the number of DSP units in the input list.
  • DSP::getNumOutputs Retrieves the number of DSP units in the output list.
  • DSP::disconnectAll Disconnects all inputs and/or outputs.
  • DSP::disconnectFrom Disconnect the specified input DSP.

Parameters:

  • DSP::getDataParameterIndex Retrieve the index of the first data parameter of a particular data type.
  • DSP::getNumParameters Retrieves the number of parameters exposed by this unit.
  • DSP::setParameterBool Sets a boolean parameter by index.
  • DSP::getParameterBool Retrieves a boolean parameter by index.
  • DSP::setParameterData Sets a binary data parameter by index.
  • DSP::getParameterData Retrieves a binary data parameter by index.
  • DSP::setParameterFloat Sets a floating point parameter by index.
  • DSP::getParameterFloat Retrieves a floating point parameter by index.
  • DSP::setParameterInt Sets an integer parameter by index.
  • DSP::getParameterInt Retrieves an integer parameter by index.
  • DSP::getParameterInfo Retrieve information about a specified parameter.

Channel format:

  • DSP::setChannelFormat Sets the PCM input format this DSP will receive when processing.
  • DSP::getChannelFormat Retrieves the PCM input format this DSP will receive when processing.
  • DSP::getOutputChannelFormat Retrieves the output format this DSP will produce when processing based on the input specified.

Metering:

  • DSP::getMeteringInfo Retrieve the signal metering information.
  • DSP::setMeteringEnabled Sets the input and output signal metering enabled states.
  • DSP::getMeteringEnabled Retrieves the input and output signal metering enabled states.

Processing:

  • DSP::setActive Sets the processing active state.
  • DSP::getActive Retrieves the processing active state.
  • DSP::setBypass Sets the processing bypass state.
  • DSP::getBypass Retrieves the processing bypass state.
  • DSP::setWetDryMix Sets the scale of the wet and dry signal components.
  • DSP::getWetDryMix Retrieves the scale of the wet and dry signal components.
  • DSP::getIdle Retrieves the idle state.

General:

  • DSP::showConfigDialog Display or hide a DSP unit configuration dialog box inside the target window.
  • DSP::reset Reset a DSPs internal state ready for new input signal.
  • DSP::release Frees a DSP object.
  • DSP::getType Retrieves the pre-defined type of a FMOD registered DSP unit.
  • DSP::getInfo Retrieves information about this DSP unit.
  • DSP::getCPUUsage Retrieves statistics on the mixer thread CPU usage for this unit.
  • DSP::setUserData Sets a user value associated with this object.
  • DSP::getUserData Retrieves a user value associated with this object.
  • DSP::getSystemObject Retrieves the parent System object.

);

DSP.getIdle(
  idle
);
idle Out

Idle state.

  • Units: Boolean

A DSP is considered idle when it stops receiving input signal and all internal processing of stored input has been exhausted.

Each DSP type has the potential to have differing idle behaviour based on the type of effect. A reverb or echo may take a longer time to go idle after it stops receiving a valid signal, compared to an effect with a shorter tail length like an EQ filter.
