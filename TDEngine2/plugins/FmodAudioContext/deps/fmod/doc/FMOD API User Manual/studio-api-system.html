6. Studio API Reference | Studio::System

The main system object for FMOD Studio.

Initializing the FMOD Studio System object will also initialize the core System object.

Lifetime:

  • Studio::System::create FMOD Studio System creation function.
  • Studio::System::initialize Initializes the Studio System.
  • Studio::System::release Shut down and free the Studio System object.
  • FMOD_STUDIO_INITFLAGS Studio System initialization flags.

Update:

  • Studio::System::update Update the FMOD Studio System.
  • Studio::System::flushCommands Block until all pending commands have been executed.
  • Studio::System::flushSampleLoading Block until all sample loading and unloading has completed.

Banks:

  • Studio::System::loadBankCustom Loads the metadata of a Studio bank using custom read callbacks.
  • Studio::System::loadBankFile Loads the metadata of a Studio bank from file.
  • Studio::System::loadBankMemory Loads the metadata of a Studio bank from memory.
  • Studio::System::unloadAll Unloads all currently loaded banks.
  • Studio::System::getBank Retrieves a loaded bank.
  • Studio::System::getBankByID Retrieves a loaded bank.
  • Studio::System::getBankCount Retrieves the number of loaded banks.
  • Studio::System::getBankList Retrieves the loaded Banks.
  • FMOD_STUDIO_BANK_INFO Information for loading a bank using user callbacks.
  • FMOD_STUDIO_LOAD_BANK_FLAGS Flags to controlling bank loading.
  • FMOD_STUDIO_LOAD_MEMORY_ALIGNMENT Required memory alignment of banks in user memory.
  • FMOD_STUDIO_LOAD_MEMORY_MODE Specifies how to use the memory buffer passed to Studio::System::loadBankMemory.

Listeners:

  • Studio::System::setListenerAttributes Sets the 3D attributes of the listener.
  • Studio::System::getListenerAttributes Retrieves listener 3D attributes.
  • Studio::System::setListenerWeight Sets the listener weighting.
  • Studio::System::getListenerWeight Retrieves listener weighting.
  • Studio::System::setNumListeners Sets the number of listeners in the 3D sound scene.
  • Studio::System::getNumListeners Retrieves the number of listeners.

Buses:

  • Studio::System::getBus Retrieves a loaded bus.
  • Studio::System::getBusByID Retrieves a loaded bus.

Events:

  • Studio::System::getEvent Retrieves an EventDescription.
  • Studio::System::getEventByID Retrieves an EventDescription.

Parameters:

  • Studio::System::getParameterByID Retrieves a global parameter value by unique identifier.
  • Studio::System::setParameterByID Sets a global parameter value by unique identifier.
  • Studio::System::setParametersByIDs Sets multiple global parameter values by unique identifier.
  • Studio::System::getParameterByName Retrieves a global parameter value by name.
  • Studio::System::setParameterByName Sets a global parameter value by name.
  • Studio::System::getParameterDescriptionByName Retrieves a global parameter by name or path.
  • Studio::System::getParameterDescriptionByID Retrieves a global parameter by ID.
  • Studio::System::getParameterDescriptionCount Retrieves the number of global parameters.
  • Studio::System::getParameterDescriptionList Retrieves a list of global parameters.

VCAs:

  • Studio::System::getVCA Retrieves a loaded VCA.
  • Studio::System::getVCAByID Retrieves a loaded VCA.

Advanced settings:

  • Studio::System::setAdvancedSettings Sets advanced settings.
  • Studio::System::getAdvancedSettings Retrieves advanced settings.
  • FMOD_STUDIO_ADVANCEDSETTINGS Settings for advanced features like configuring memory and cpu usage.

Command capture and replay:

  • Studio::System::startCommandCapture Recording Studio commands to a file.
  • Studio::System::stopCommandCapture Stop recording Studio commands.
  • Studio::System::loadCommandReplay Load a command replay.
  • FMOD_STUDIO_COMMANDCAPTURE_FLAGS Flags controling command capture.
  • FMOD_STUDIO_COMMANDREPLAY_FLAGS Flags controlling command replay.

Profiling:

  • Studio::System::getBufferUsage Retrieves buffer usage information.
  • Studio::System::resetBufferUsage Resets memory buffer usage statistics.
  • Studio::System::getCPUUsage Retrieves performance information.
  • Studio::System::getMemoryUsage Retrieves memory usage statistics.
  • FMOD_STUDIO_BUFFER_INFO Information for a single buffer in FMOD Studio.
  • FMOD_STUDIO_BUFFER_USAGE Information for FMOD Studio buffer usage.
  • FMOD_STUDIO_CPU_USAGE Performance information for FMOD Studio and core systems.

Plugins:

  • Studio::System::registerPlugin Registers a plugin DSP.
  • Studio::System::unregisterPlugin Unregisters a plugin DSP.

System callback:

  • Studio::System::setUserData Sets the user data.

cbsize
Size of this structure in bytes. Must be set to sizeof(FMOD_STUDIO_ADVANCEDSETTINGS) before calling Studio::System::setAdvancedSettings or Studio::System::getAdvancedSettings.
commandqueuesize

Command queue size for studio async processing.

  • Units: Bytes
  • Default: 32768
handleinitialsize

Initial size to allocate for handles. Memory for handles will grow as needed in pages.

  • Units: Bytes
  • Default: 8192 * sizeof(void*)
studioupdateperiod

Update period of Studio when in async mode, in milliseconds. Will be quantized to the nearest multiple of mixer duration.

  • Units: Milliseconds
  • Default: 20
idlesampledatapoolsize

Size in bytes of sample data to retain in memory when no longer used, to avoid repeated disk I/O. Use -1 to disable.

  • Units: Bytes
  • Default: 262144
streamingscheduledelay

Specify the schedule delay for streams, in samples. Lower values can reduce latency when scheduling events containing streams but may cause scheduling issues if too small.

  • Units: Samples
  • Default: 8192
encryptionkey
Specify the key for loading sounds from encrypted banks. (UTF-8 string)

When calling Studio::System::setAdvancedSettings any member other than cbsize may be set to zero to use the default value for that setting.

When mode is FMOD_STUDIO_LOAD_MEMORY FMOD will allocate an internal buffer and copy the data from the passed in buffer before using it. When used in this mode there are no alignment restrictions on buffer and the memory pointed to by buffer may be cleaned up at any time after this function returns.

You can ensure the memory is not being freed prematurely by only freeing it after receiving the FMOD_STUDIO_SYSTEM_CALLBACK_BANK_UNLOAD callback.

By default this function will block until the load finishes and return the FMOD_RESULT indicating the result. If the load fails then bank will contain NULL.

Using the FMOD_STUDIO_LOAD_BANK_NONBLOCKING flag will cause the bank to be loaded asynchronously. In that case this function will always return FMOD_OK and bank will contain a valid bank handle. Load errors for asynchronous banks can be detected by calling Studio::Bank::getLoadingState. Failed asynchronous banks should be released by calling Studio::Bank::unload.

This function is not compatible with FMOD_STUDIO_ADVANCEDSETTINGS::encryptionkey, using them together will cause an error to be returned.

See Also: Studio::System::loadBankFile, Studio::System::loadBankCustom, Studio::System::setCallback

Studio::System::update

Update the FMOD Studio System.

C
C++
C#
JS

FMOD_RESULT Studio::System::update();

FMOD_RESULT FMOD_Studio_System_Update(FMOD_STUDIO_SYSTEM *system);

RESULT Studio.System.update();

Studio.System.update();

When Studio is initialized in the default asynchronous processing mode this function submits all buffered commands for execution on the Studio Update thread for asynchronous processing. This is a fast operation since the commands are not processed on the calling thread. If Studio is initialized with FMOD_STUDIO_INIT_DEFERRED_CALLBACKS then any deferred callbacks fired during any asynchronous updates since the last call to this function will be called. If an error occurred during any asynchronous updates since the last call to this function then this function will return the error result.

When Studio is initialized with FMOD_STUDIO_INIT_SYNCHRONOUS_UPDATE queued commands will be processed immediately when calling this function, the scheduling and update logic for the Studio system are executed and all callbacks are fired. This may block the calling thread for a substantial amount of time.
