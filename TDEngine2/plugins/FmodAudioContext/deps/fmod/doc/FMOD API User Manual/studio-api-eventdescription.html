6. Studio API Reference | Studio::EventDescription

The description for an FMOD Studio Event.

Event descriptions belong to banks and can be queried after the relevant bank has been loaded. Event descriptions may be retrieved via path or GUID lookup, or by enumerating all descriptions in a bank.

Instances:

  • Studio::EventDescription::createInstance Creates a playable instance.
  • Studio::EventDescription::getInstanceCount Retrieves the number of instances.
  • Studio::EventDescription::getInstanceList Retrieves a list of the instances.
  • Studio::EventDescription::releaseAllInstances Releases all instances.

Sample Data:

  • Studio::EventDescription::loadSampleData Loads non-streaming sample data used by the event.
  • Studio::EventDescription::unloadSampleData Unloads all non-streaming sample data.
  • Studio::EventDescription::getSampleLoadingState Retrieves the sample data loading state.

Attributes:

  • Studio::EventDescription::is3D Retrieves the event's 3D status.
  • Studio::EventDescription::isOneshot Retrieves the event's oneshot status.
  • Studio::EventDescription::isSnapshot Retrieves the event's snapshot status.
  • Studio::EventDescription::isStream Retrieves the event's stream status.
  • Studio::EventDescription::hasCue Retrieves whether the event has any sustain points.
  • Studio::EventDescription::getMaximumDistance Retrieves the maximum distance for 3D attenuation.
  • Studio::EventDescription::getMinimumDistance Retrieves the minimum distance for 3D attenuation.
  • Studio::EventDescription::getSoundSize Retrieves the sound size for 3D panning.

Parameters:

  • Studio::EventDescription::getParameterDescriptionByName Retrieves an event parameter description by name.
  • Studio::EventDescription::getParameterDescriptionByID Retrieves an event parameter description by id.
  • Studio::EventDescription::getParameterDescriptionByIndex Retrieves an event parameter description by index.
  • Studio::EventDescription::getParameterDescriptionCount Retrieves the number of parameters in the event.

User Properties:

  • Studio::EventDescription::getUserProperty Retrieves a user property by name.
  • Studio::EventDescription::getUserPropertyByIndex Retrieves a user property by index.
  • Studio::EventDescription::getUserPropertyCount Retrieves the number of user properties attached to the event.
  • FMOD_STUDIO_USER_PROPERTY Describes a user property.
  • FMOD_STUDIO_USER_PROPERTY_TYPE User property types.

General:

  • Studio::EventDescription::getID Retrieves the GUID.
  • Studio::EventDescription::getLength Retrieves the length of the timeline.
  • Studio::EventDescription::getPath Retrieves the path.
  • Studio::EventDescription::getUserData Retrieves the event user data.
  • Studio::EventDescription::setCallback Sets the user callback.
  • Studio::EventDescription::setUserData Sets the event user data.
  • Studio::EventDescription::isValid Checks that the EventDescription reference is valid.

See Also:

  • Studio::System::getEvent Retrieves an EventDescription.
  • Studio::System::getEventByID Retrieves an EventDescription.
  • Studio::Bank::getEventList Retrieves a list of the event descriptions in the bank.

C

JS

FMOD_RESULT Studio::EventDescription::is3D(
  bool *is3d
);

FMOD_RESULT FMOD_Studio_EventDescription_Is3D(
  FMOD_STUDIO_EVENTDESCRIPTION *eventdescription,
  FMOD_BOOL *is3d
);

RESULT Studio.EventDescription.is3D(
  out bool is3d
);

Studio.EventDescription.is3D(
  is3d
);
is3d Out

3D status. True if the event is 3D.

  • Units: Boolean

An event is considered 3D if any of these conditions are met:

  • The event has a Spatializer, 3D Object Spatializer, or a 3rd party spatializer on its master track.
  • The event contains an automatic parameter that depends on the event's 3D attributes:
    • Distance
    • Event Cone Angle
    • Event Orientation
    • Direction
    • Elevation
  • The event contains any nested events which are 3D.

Note: If the event contains nested events built to separate banks using versions of FMOD Studio prior to 2.00.10 and those banks have not been loaded then this function may fail to correctly determine the event's 3D status.
