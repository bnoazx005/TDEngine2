6. Studio API Reference | Studio::Bank

FMOD Studio banks contain the metadata and audio sample data required for runtime mixing and playback. Audio sample data may be packed into the same bank as the events which reference it, or it may be packed into separate banks.

Loading:

  • Studio::Bank::getLoadingState Retrieves the loading state.
  • Studio::Bank::loadSampleData Loads non-streaming sample data for all events in the bank.
  • Studio::Bank::unloadSampleData Unloads non-streaming sample data for all events in the bank.
  • Studio::Bank::getSampleLoadingState Retrieves the loading state of the samples in the bank.
  • Studio::Bank::unload Unloads the bank.

Buses:

  • Studio::Bank::getBusCount Retrieves the number of buses in the bank.
  • Studio::Bank::getBusList Retrieves a list of the buses in the bank.

Events:

  • Studio::Bank::getEventCount Retrieves the number of event descriptions in the bank.
  • Studio::Bank::getEventList Retrieves a list of the event descriptions in the bank.

String Table:

  • Studio::Bank::getStringCount Retrieves the number of string table entries in the bank.
  • Studio::Bank::getStringInfo Retrieves a string table entry.

VCAs:

  • Studio::Bank::getVCACount Retrieves the number of VCAs in the bank.
  • Studio::Bank::getVCAList Retrieves a list of the VCAs in the bank.

General:

  • Studio::Bank::getID Retrieves the GUID.
  • Studio::Bank::getPath Retrieves the path.
  • Studio::Bank::setUserData Sets the bank user data.
  • Studio::Bank::getUserData Retrieves the bank user data.
  • Studio::Bank::isValid Checks that the Bank reference is valid.

See Also:

  • Studio::System::loadBankFile Loads the metadata of a Studio bank from file.
  • Studio::System::loadBankMemory Loads the metadata of a Studio bank from memory.
  • Studio::System::loadBankCustom Loads the metadata of a Studio bank using custom read callbacks.
  • Studio::System::getBank Retrieves a loaded bank.
  • Studio::System::getBankById Retrieves a loaded bank.
  • Studio::System::getBankList Retrieves the loaded Banks.

  state
);
state Out
Loading state. (FMOD_STUDIO_LOADING_STATE)

This function may be used to check the loading state of a bank which has been loaded asynchronously using the FMOD_STUDIO_LOAD_BANK_NONBLOCKING flag, or is pending unload following a call to Studio::Bank::unload.

If an asynchronous load failed due to a file error state will contain FMOD_STUDIO_LOADING_STATE_ERROR and the return code from this function will be the error code of the bank load function.

See Also: Studio::System::loadBankFile, Studio::System::loadBankMemory, Studio::System::loadBankCustom

Studio::Bank::unload

Unloads the bank.

C
C++
C#
JS

FMOD_RESULT Studio::Bank::unload();

FMOD_RESULT FMOD_Studio_Bank_Unload(FMOD_STUDIO_BANK *bank);

RESULT Studio.Bank.unload();

Bank.unload();

This will destroy all objects created from the bank, unload all sample data inside the bank, and invalidate all API handles referring to the bank.

If the bank was loaded from user-managed memory, e.g. by Studio::System::loadBankMemory with the FMOD_STUDIO_LOAD_MEMORY_POINT mode, then the memory must not be freed until the unload has completed. Poll the loading state using Studio::Bank::getLoadingState or use the FMOD_STUDIO_SYSTEM_CALLBACK_BANK_UNLOAD system callback to determine when it is safe to free the memory.

See Also: Studio::System::loadBankFile, Studio::System::loadBankCustom, Studio::System::setCallback
